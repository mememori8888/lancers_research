## 1. プロジェクト概要

本プロジェクトは、**ISO 30414:2018（人的資本報告ガイドライン）** に基づく63種類の指標を、各企業の公開PDFレポートから自動抽出し、横断比較可能な形式（CSV）に集約することを目的としています。

### ISO 30414 とは

ISO 30414は、組織の人的資本に関する情報を外部・内部に開示するための国際標準（International Organization for Standardization）であり、以下の領域をカバーします。

| 領域 | 主な指標例 |
|------|-----------|
| コンプライアンス・倫理 | 苦情件数、懲戒処分件数、コンプライアンス研修受講率 |
| コスト | 総労働力コスト、採用コスト、研修費用 |
| ダイバーシティ | 年齢構成比、男女比率 |
| リーダーシップ・エンゲージメント | エンゲージメントスコア、研修参加率 |
| 安全衛生 | 労災損失時間、死亡者数 |
| 生産性 | 従業員1人あたり売上・EBIT・純利益、人的資本ROI |
| 採用・後継者計画 | 採用日数、内部登用率 |
| 離職 | 離職率、自発的離職率 |
| 研修・開発 | 研修時間、参加率 |
| 従業員数 | 総従業員数、FTE、派遣労働者数 |

## 3. 処理パイプライン（3ステージ構成）

本システムは以下の3ステージで動作します。

```
[Stage 1]  PDFから変換済みのMarkdown (.md) ファイル
               ↓  Gemini 2.5 Flash による構造化解析
           AIキャッシュ JSON (*.md.gemini_v1.json)

[Stage 2]  AIキャッシュ JSON を全企業分ロード
               ↓  スコープ補正 → 衝突解決 → 集約
           行データ（企業 × 指標 × 年度）

[Stage 3]  行データ → CSV 出力
           aggregated_iso30414_ai.csv
```

### 3.1 Stage 1：Markdown → JSON キャッシュ（Gemini による解析）

各MDファイルを **Google Gemini 2.5 Flash** に送り、以下のJSONスキーマで構造化データを返します。

```json
{
  "document_scope": "consolidated | standalone | unknown",
  "metrics": [
    {
      "metric_id": "指標ID",
      "year_values": {
        "2023": {
          "value": "913.9",
          "unit": "EUR million",
          "page": "247"
        }
      }
    }
  ]
}
```

- キャッシュは `<mdファイルパス>.gemini_v1.json` として保存
- 再実行時は `--overwrite` フラグで再生成
- レートリミット対策として最大6回リトライ、指数バックオフを実装

### 3.2 Stage 2：集約・衝突解決

1. 全企業のキャッシュJSONを読み込み
2. ファイル名キーワードで `document_scope` を補正（後述）
3. 同一（企業・指標・年度）に複数ソースが存在する場合は **スコープ優先ルール** で採用値を決定
4. 行データとして出力

### 3.3 Stage 3：CSV 出力

- 出力形式: UTF-8 BOM付き CSV（日本語Excelでの直接表示に対応）
- 区切り文字: `,`（カンマ）
- 年度列: 2014〜2025年度（12列）

## 4. 連結・単体の判定基準

「連結 or 単体」列は以下の2段階で判定しています。

### 4.1 ファイル名キーワードによる補正（優先）

ファイル名に以下のパターンが含まれる場合、AIの判定によらず **強制的に `standalone`（単体）** とします。

| パターン | 対象文書例 | 理由 |
|---------|-----------|------|
| `Jahresabschluss` | DWS KGaA Jahresabschluss 2021 | 独語：法人単体の年次決算書 |
| `Einzelabschluss` | Einzelabschluss Infineon AG 2022 | 独語：個別財務諸表 |
| `KGaA Report` | DWS KGaA Report 2023 | 持株会社（KGaA）単体の事業報告書 |
| `KGaA Bericht` | （同上・独語版） | 同上 |
| `Standalone` | — | 英語での明示的な単体表記 |
| `Separate financial` | Separate financial statements | 英語：個別財務諸表 |

> **この補正が必要な背景**: AIは「KGaA Report」を誤って `consolidated`（連結）と判定することがありました。
> KGaA（株式合資会社）は親会社の持株会社形式であり、その単独報告書は **単体** に分類されます。

### 4.2 AI（Gemini）によるスコープ判定

ファイル名での補正が適用されない場合、Stage 1でGeminiが文書内容を読んで以下のルールで判定します。

| 判定結果 | 条件 |
|---------|------|
| `consolidated`（連結） | "Group"、"consolidated"、"Konzernabschluss"等が含まれる<br>グループ全体を対象とした Annual Report |
| `standalone`（単体） | "Jahresabschluss"、"KGaA Report"、"standalone"等が含まれる |
| `unknown`（不明） | 上記で判断できない場合 |

### 4.3 出力列への反映

採用された全年度のスコープを集計し、以下の値を出力します。

| 出力値 | 条件 |
|--------|------|
| 連結 | 全年度が `consolidated` のみ |
| 単体 | 全年度が `standalone` のみ |
| 連結・単体混在 | 年度によって `consolidated` と `standalone` が混在 |
| （空欄） | 全年度が `unknown`（スコープ特定不能） |

> **重要**: スコープが `unknown` のファイルは「連結 or 単体」列に記載しません。
> 推測での記入は行わず、確認可能な情報のみを記録します。

## 5. 金額系指標の品質管理

### 5.1 ルール

**金額系指標**（通貨単位 EUR / USD / JPY 等を持つ指標、例: `sales_per_employee`, `total_labor_cost`）については、**正式アニュアルレポート以外のソースを除外** します。

#### 除外されるソース（金額系のみ）

| ソース種別 | 除外理由 |
|-----------|---------|
| KGaA Report / Jahresabschluss | 連結全体でなく持株会社単体の数値のため、スケールが大幅に異なる |
| メディア向け資料 / プレスリリース | 概算値（`~5,000名`等）が含まれ、精度が低い |
| IR 決算発表スライド（単体） | 部分集計や速報値が含まれる場合がある |

#### 使用されるソース（金額系）

- `Annual Report`（年次報告書）のみ
- ただし `Annual Report` であっても `Annual Report Media Info` 等、メディア向け名称が含まれるファイルは除外

### 5.2 この制約が重要な理由（例：DWS）

DWSの `sales_per_employee`（従業員一人当たり売上）を例に挙げます。

| ソースファイル | 区分 | 従業員数 | 金額 | 算出値 |
|-------------|------|---------|------|--------|
| DWS Annual Report 2021 | **連結**（正） | 3,422 FTE | €2,720M | **€794千/FTE** |
| DWS KGaA Report 2022 | 単体（除外） | 173名 | €75M | €434千（比較不能） |
| Media info 2026 | 除外 | ~5,000名（概算） | — | 信頼性低 |

連結と単体では **従業員数が約20倍異なる** ため、混在すると値が全く意味をなさなくなります。
この制約により、年度ごとに正式 Annual Report が存在しない場合は **空欄** とします。

## 6. 複数ソース間の値衝突への対応

同一（企業・指標・年度）に複数ファイルから異なる値が抽出された場合の処理です。

### 6.1 解決フロー

```
Step 1: スコープ優先フィルタ
        consolidated(2) > unknown(1) > standalone(0)
        最高優先グループのエントリのみ残す

Step 2: 最高優先グループ内で値を比較
        → 一致: そのまま採用（「連結優先（standaloneを除外）」とメモ）
        → 不一致: 最初の値を仮採用し「要確認（値一←ソース / 値二←ソース）」とメモ
```

### 6.2 メモ欄での表示例

| メモ例 | 意味 |
|--------|------|
| `2021年: 連結優先（standaloneを除外）` | 連結値を優先採用、単体値は除外 |
| `2021年: 要確認（76←レポートA p.41 / 79←レポートB p.55）` | 同一スコープ内で値が異なる。手動確認推奨 |
| （空欄） | 1つのソースのみ、または完全一致 |

### 6.3 「要確認」フラグが立つ主な原因

1. **集計対象範囲の違い**：サステナビリティレポートと Annual Report で対象範囲が異なる
2. **記載時点の違い**：前年度レポートと当該年度レポートで確定値と速報値が異なる
3. **定義の違い**：エンゲージメントスコアを%で示すレポートと100点満点で示すレポートがある
4. **集計単位の違い**：法人単体 vs グループ子会社合算

## 9. 注意事項・制限事項

### 9.1 AI解析の限界

- Gemini 2.5 Flashは高精度ですが、**100%の正確性は保証できません**
- 特に以下の場合は誤抽出・誤判定のリスクがあります：
  - 表の縦横が複雑に入り組んでいる箇所
  - 同一ページに複数年度・複数指標が密集している箇所
  - ドキュメントのレイアウトが特殊な場合（横スキャン・グラフ内数値等）

### 9.2 「要確認」フラグのある行

メモ欄に「要確認」と記載されている行は、**複数ソースで値が一致しない** ことを示します。  
採用値は最初に見つかった値（スコープ優先グループの先頭）であり、  
最終的な確定値とするには原典資料との照合を推奨します。

### 9.3 単位の統一

- 金額の単位（EUR thousand / EUR million 等）は **AIが抽出した単位を優先** します
- 同一企業・同一指標で年度によって単位が変わる場合、集約時に最頻値を採用します
- 単位欄が空欄の場合は、AIが単位を特定できなかったことを意味します

### 9.4 Infineon の会計年度

Infineon は **10月末決算**（例: FY2023 = 2022年10月〜2023年9月）のため、  
他社の「2023年度」（1〜12月）とは期間が異なります。  
比較の際はこの点に留意してください。

### 9.5 「連結 or 単体」が空欄の行

空欄は「スコープが特定できなかった」ことを意味し、値の信頼性が低い可能性があります。  
「推測で連結にする」等の操作は行っていません。

## 10. 用語集

| 用語 | 説明 |
|------|------|
| **ISO 30414** | 人的資本に関する情報開示の国際標準（2018年制定） |
| **連結（consolidated）** | グループ全体（親会社＋子会社）を合算した財務・非財務情報 |
| **単体（standalone）** | 親会社法人単独の財務情報。グループ子会社を含まない |
| **FTE（Full-Time Equivalent）** | フルタイム換算従業員数。パートタイムをフルタイム換算で合算した値 |
| **EBIT** | 利息・税引前利益（Earnings Before Interest and Taxes） |
| **KGaA** | ドイツ法人形態：株式合資会社（Kommanditgesellschaft auf Aktien） |
| **Jahresabschluss** | ドイツ語：年次決算書（主に法人単体の財務諸表） |
| **Einzelabschluss** | ドイツ語：個別財務諸表（= standalone financial statements） |
| **Gemini 2.5 Flash** | Google の大規模言語モデル。Stage 1 のPDF解析に使用 |
| **要確認フラグ** | メモ欄に記載。複数ソースで値が異なる場合に自動付与 |
| **アニュアルレポートフィルタ** | 金額系指標において、正式 Annual Report 以外のソースを除外するルール |
